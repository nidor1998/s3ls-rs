# `--max-depth` Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `--max-depth <N>` CLI option that limits recursive listing depth at the S3 API level, preventing objects beyond the specified depth from being fetched.

**Architecture:** The option flows from CLI args through Config to the ListingEngine. In parallel listing mode, recursion stops at max-depth by listing with delimiter `"/"` and not following sub-prefixes. In sequential fallback mode, objects are filtered by counting `/` separators relative to the listing prefix.

**Tech Stack:** Rust, clap (CLI parsing), tokio (async), aws-sdk-s3

- [ ] **Step 3: Add depth calculation helper**

Add a helper method to `ListingEngine` (before `list_dispatch`):

```rust
    /// Calculate the depth of a key relative to the engine's prefix.
    /// Depth = (number of "/" in the relative part of the key) + 1.
    /// Returns None if the key doesn't start with the prefix.
    fn key_depth(&self, key: &str) -> Option<u16> {
        let prefix = self.prefix.as_deref().unwrap_or("");
        let relative = key.strip_prefix(prefix)?;
        if relative.is_empty() {
            return None; // key IS the prefix, not an object under it
        }
        let slash_count = relative.matches('/').count();
        // "file.txt" => 0 slashes => depth 1
        // "a/file.txt" => 1 slash => depth 2
        Some(slash_count as u16 + 1)
    }
```

- [ ] **Step 4: Add depth filtering to `send_listed_entries`**

Replace the `send_listed_entries` method with a version that respects `max_depth` when set:

```rust
    /// Send a batch of entries to the channel, returning `true` if sending
    /// should stop (cancellation or receiver dropped).
    /// When `max_depth` is set, entries beyond the depth limit are silently
    /// dropped (used as fallback when the parallel path couldn't prevent
    /// fetching them).
    async fn send_listed_entries(
        &self,
        entries: Vec<ListEntry>,
        sender: &Sender<ListEntry>,
    ) -> Result<bool> {
        for entry in entries {
            if self.cancellation_token.is_cancelled() {
                return Ok(true);
            }
            // Apply max_depth filter for objects (CommonPrefix and DeleteMarker
            // entries also carry a key that can be depth-checked).
            if let Some(max_depth) = self.max_depth {
                if let Some(depth) = self.key_depth(entry.key()) {
                    if depth > max_depth {
                        continue;
                    }
                }
            }
            if sender.send(entry).await.is_err() {
                return Ok(true); // receiver dropped
            }
        }
        Ok(false)
    }
```
